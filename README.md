# transcendence

## 프로젝트 목표
  - Single Page Application
  - OAuth 2.0 authentication with 42
  - 유저 인터페이스를 통한 친구 기능 및 프로필
  - 실시간 채팅
  - 실시간 Pong game

## 기술 스택
  - FE: React
  - BE: nestJs
  - DB: postgresql

## 역할 
3인 프로젝트를 진행했고, 

개발으로 게임 구현과 초대기능, 채팅환경, 그리고 화면단의 전체적인 부분을 구현하였고 
화면의 변경과, 유저의 정보를 가져오고 가져온 정보에서 다른 유저의 접속기록과 관련된 부분을 개발하였습니다.
백엔드 처리에서 게임로직에서 데이터를 보내주고 받아오며 데이터를 세팅하는 부분을 구현하게 되었습니다.
백엔드 팀원과 프론트 팀원간의 의사소통 부분을 담당하여 필요한 부분들을 대부분 맡아 처리 하였고 
소통을 위해 정리하여 요구사항을 분석하는 역할을 담당하였습니다.
백엔드에서 데이터를 받아오고 하는 부분에 처리가 안되는것들을 해결하는 일또한 하게 되었습니다.
실제 평가를 위한 도커컴포즈에서 프론트의 처리 부분을 담당하게 되었습니다.

## 시행착오
-  실시간 채팅 및 게임 구현: socketIO를 통해 구현했습니다. 처음에는 각자 코드를 작성했지만 막히는 부분이 많았고 백엔드를 맡은 동료와 협업을 통해 해결했습니다.
-  api호출: cors에러등 여러가지 문제점이 많았지만 axios 세팅을 통해 해결할 수 있었습니다.
-  전역변수: 페이지 이동할 때마다 api호출을 하는건 비효율적이라 생각해서 zustand를 이용한 전역변수 세팅을 해주었고, 새로고침이 될 때 api호출을 통해 해결했습니다.

# 느낀점
-  팀원간의 갈등이 발생하여 코드를 구현하다, 프로젝트가 실패할 뻔 하였고,이를 중재하면서 사람간의 관계가 참 중요하다는것을 알수 있었습니다.
